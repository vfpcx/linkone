## 4. 风险登记

| # | 风险 | 影响 | 缓解 |
|---|---|---|---|
| R-01 | 试点仓库不愿意启用批次管理，导致 FIFO/临期数据缺失 | 中 | 默认关闭即兼容；批次启用后再宣传价值 |
| R-02 | 语音识别率不达预期 | 中 | 表单始终可手填；提供"放弃录音"兜底 |
| R-03 | 容量公示给全平台批发商引发竞品挖角 | 中 | TA 自主选 3 档可见性；可选模糊档脱敏 |
| R-04 | OPS 代建租户/批发商被滥用（绕过审核） | 中 | 操作日志强留痕 + 二次复核机制 + 来源标记 |
| R-05 | 地图 SDK API 调用量超预算 | 低 | 容量公示用 10 分钟快照、距离计算前端缓存 |
| R-06 | 多租户数据隔离漏洞（tenant_id 缺失） | 高 | 架构师补强：DAO 层强制注入 tenant_id；测试覆盖 |
| R-07 | 平台不接资金但客户期望平台担保 | 中 | 客诉仲裁机制 + 黑名单 + 用户教育 |
| R-08 | 终端隐私（位置/手机号）保护不足 | 高 | 法务把关 + 注销冷静期 + 本地缓存 |
