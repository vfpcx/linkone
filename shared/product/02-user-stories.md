## 0. 故事编写约定

- **格式**：`作为 [角色]，我希望 [动作]，以便 [价值]`
- **优先级**：P0 = MVP 必做 · P1 = 试点期内迭代 · P2 = 试点后再排
- **验收**：每个故事给出 ≥1 条可观察的验收点（Given/When/Then 简写）
- **编号**：US-[角色缩写]-[序号]，便于跨文档引用

角色缩写：OPS / TA / WK / ST / WA / WE / RT

## 0.5 通用账号能力（跨角色）

> 适用于所有可登录角色：OPS / TA / WK / ST / WA / WE / RT。
> RT 默认采用「手机号 + 短信验证码」无密码登录；其他角色必须有密码。

### US-COMMON-01 · P0 · 注册（角色感知）
**作为** 任意未注册用户，**我希望** 按不同入口注册账号，**以便** 进入平台。

**注册入口与归属**：
| 入口 | 归属角色 | 来源 |
|---|---|---|
| 平台首页「冻库/仓库入驻」公开页 | TA | 自助注册（待 OPS 审核）|
| OPS 后台「新建租户」 | TA | 代建（无需 TA 自助）|
| TA 后台「员工注册码」扫码 | WK / ST | 受邀注册 |
| 仓库广场「申请入驻」 | WA | 自助提交申请（待 TA 审批）|
| OPS 后台「代建批发商」 | WA | 代建（需 TA 授权或客诉联动）|
| WA 后台「员工注册码」扫码 | WE | 受邀注册 |
| 任意 RT 入口的"登录/注册" | RT | 手机号 + 验证码即注册 |

**字段约定**：
- 手机号（必填，全平台唯一标识）
- 短信验证码（必填）
- 密码（除 RT 外必填，6-20 位含数字+字母）
- 真实姓名（员工/管理员必填）
- 同意《用户协议》《隐私政策》

**验收**：
- 同手机号已在该角色域内存在 → 提示"该手机号已注册，请直接登录或找回密码"
- 同手机号在不同角色域可重复存在（如某号既是某仓库 TA，又是另一仓库的 WA），但每一行 UserRole 独立
- OPS 代建场景的临时密码须强制首登修改（US-OPS-01b D31）
- 短信验证码 5 分钟有效，60 秒内不可重发

### US-COMMON-02 · P0 · 登录
**作为** 已注册用户，**我希望** 选择多种方式登录，**以便** 快速进入工作台。

**登录方式**：
| 方式 | 适用角色 | 备注 |
|---|---|---|
| 手机号 + 密码 | OPS / TA / WK / ST / WA / WE | 默认方式 |
| 手机号 + 短信验证码 | 所有角色（含 RT）| 找回密码 / 不便输入密码场景 |
| 微信扫码（v2 评估） | TA / WA / RT | MVP 不做，预留入口 |

**多角色处理**：
- 登录成功后若该手机号绑定多个 UserRole（如 TA + WA），弹出"切换工作空间"选择器
- 同一手机号在同租户内的多个内部角色（TA + WK + ST 兼任）登录后自动取并集权限（US-TA-06）

**安全规则**：
- 5 次密码错误锁定账号 30 分钟
- 登录 IP 异常时强制短信二次校验（v2，MVP 仅做 IP 日志）
- 30 天未操作自动退出登录

**验收**：
- 登录页支持两种方式 Tab 切换
- 登录后跳转该用户主角色的工作台（按 TA > ST > WK > WA > WE 优先级）
- 失败提示统一为"账号或密码错误"，不暴露账号是否存在

### US-COMMON-03 · P0 · 修改密码
**作为** 已登录用户（除 RT 外），**我希望** 在「我的 → 安全设置」修改密码，**以便** 保护账号。

**流程**：
1. 输入原密码（验证）
2. 输入新密码 × 2（前端校验：6-20 位含数字+字母）
3. 短信验证码二次确认（防原密码泄漏）
4. 修改成功 → 自动退出全部已登录设备 → 跳登录页

**验收**：
- 修改成功 1 分钟内所有 token 失效
- 操作日志留痕（时间、IP、设备）
- 90 天未修改密码可在工作台弹"建议修改密码"轻提示（不强制）
- 不支持新密码与最近 3 次相同

### US-COMMON-04 · P0 · 找回密码
**作为** 任意可登录用户（除 RT 外），**我希望** 通过手机号短信找回密码，**以便** 在忘记密码时恢复访问。

**流程**：
1. 登录页点「忘记密码」
2. 输入注册手机号 + 短信验证码
3. 设置新密码 × 2（同 US-COMMON-03 强度要求）
4. 重置成功 → 跳登录页 + 退出全部已登录设备

**安全限制**：
- 同手机号 24h 内最多 5 次找回密码请求
- 短信验证码 5 分钟有效
- 重置成功后短信通知用户"您的密码已被重置，如非本人操作请联系客服"

**验收**：
- 未注册手机号请求时仍返回"验证码已发送"（不暴露账号存在性，但实际不发）
- 操作日志留痕

### US-COMMON-05 · P0 · 换绑手机号
**作为** 已登录用户，**我希望** 把账号绑定的手机号换成新号，**以便** 应对换号场景。

**流程**：
1. 「我的 → 安全设置 → 换绑手机号」
2. 验证原手机号 + 验证码
3. 输入新手机号 + 新手机号验证码
4. 输入登录密码（RT 除外）二次确认
5. 完成换绑 → 操作日志留痕

**验收**：
- 新手机号若已被同角色域账号占用 → 拒绝换绑 + 提示"新号已注册"
- 换绑后所有 token 失效，需重新登录
- 历史业务数据（询价/单据/账单）保留，不迁移

### US-COMMON-06 · P0 · 验证码登录与注册（RT 专属轻路径）
**作为** 终端买家（RT），**我希望** 用「手机号 + 短信验证码」一步登录，未注册时自动注册，**以便** 最快下单/询价。

**流程**：
1. 扫店铺码或访问平台首页，点击"询价/我的"
2. 输入手机号 + 验证码
3. 若新号 → 自动创建 RT 账号 + 默认昵称（"用户****1234"）+ 同意协议复选框
4. 若老号 → 直接登录
5. 不强制设置密码；RT 可在「我的 → 安全设置」可选启用密码登录（默认不启用）

**与 R16 RT 注销联动**：
- 注销冷静期内（7 天）尝试用同手机号"登录" → 弹"账号正在注销中，是否撤销注销？"
- 7 天后该手机号视为已删除，再注册即新账号

**验收**：
- RT 注册无需姓名、密码、身份证
- 默认昵称可在"我的 → 个人资料"修改
- 短信验证码与其他角色共享同套限频规则

### US-COMMON-07 · P0 · 退出登录
**作为** 已登录用户，**我希望** 主动退出登录，**以便** 共用设备时保护账号。
**验收**：「我的 → 退出登录」 → 当前设备 token 失效 → 跳登录页；不影响其他设备的 token。

## 1. 平台运维（OPS）

### US-OPS-01 · P0 · 仓库入驻审核
**作为** 平台运维，**我希望** 在后台看到待审核的仓库入驻申请列表，逐条查看资质材料并通过/驳回，**以便** 控制平台入驻质量。
**验收**：通过后 1 分钟内对应仓库可登录后台并完成店铺初始化；驳回需填写理由并短信通知。

### US-OPS-00 · P0 · 地址需地图选点 + 坐标
**作为** 平台所有角色，**我希望** 录入或展示「地址」时统一通过地图组件选点（输出坐标 lng/lat）并支持地图浏览，**以便** 后续基于坐标做距离计算、仓库推荐、收货半径校验等。

**适用入口（统一组件）**：
| 入口 | 角色 | 用途 |
|---|---|---|
| OPS 代建租户的「仓库地址」 | OPS | 地图选点，落库 lng/lat |
| TA 修改店铺基础资料的「仓库地址」 | TA | 地图选点 / 改点 |
| WA 入驻申请前查看「仓库位置」 | WA | 只读地图浏览 |
| RT 提交意向单的「收货地址」 | RT | 地图选点 / 用我当前位置 / 用历史地址 |
| ST 已收款登记的地址追溯 | ST | 只读地图浏览 |
| 仓库广场卡片的「附近仓库」 | WA / RT | 地图列表混合模式 |

**字段约定**：
- 文本地址（用户可读，地图反查或手工补充）
- lng / lat（数值坐标，WGS84 或 GCJ-02 选一统一，详见 05 §15）
- 精度等级 / 来源（GPS 定位 / 地图选点 / 输入反查 / IP 兜底）

**降级**：
- 用户拒绝定位 → 弹"请手动在地图选点或输入地址"
- 地图 SDK 加载失败 → 退回纯文本地址输入 + 标注"未获取坐标，部分推荐功能不可用"

**验收**：
- 全平台所有"地址"字段均输出 (text, lng, lat, source) 四元组
- 任意角色查看带坐标的地址记录时可在地图上一键查看
- 坐标精度低于阈值（如 IP 定位 100km 以上）时标注"位置粗略"

### US-RT-06 · P0 · 基于位置推荐仓库
**作为** 二批/终端，**我希望** 进入平台首页时基于我的当前位置（或上次定位）智能排序仓库列表，**以便** 优先看到附近仓库的商品。

**推荐策略**：
1. 用户首次进入 → 请求定位权限
   - 同意 → 用 GPS 坐标
   - 拒绝 → 用 IP 粗略定位 / 让用户手动选城市
   - 历史用户 → 用上次缓存的坐标（≤7 天有效）
2. 列表排序：默认按"距离我"升序，可切换"剩余容量降序 / 推荐综合"
3. 距离展示：< 1 km 显米，≥1 km 显公里，> 50 km 折叠到"远距离仓库"分区
4. 与「容量公示策略」联动：不公开仓库不进入推荐列表

**验收**：
- 首次访问弹定位授权（一次性）
- 列表第一屏 ≥ 80% 是 ≤ 20 km 内的仓库（如有）
- 拒绝定位/无 GPS → 列表按"推荐综合"排序，提示"开启定位获得更准推荐"
- 切换城市后列表立即按新位置重排
- 用户隐私：仅在客户端本地缓存最近坐标，不持久化到 RT 个人资料

### US-WA-01c · P0 · 入驻前地图查看仓库位置
**作为** 批发商管理员，**我希望** 在「仓库广场」浏览仓库时点击地图按钮看到仓库实际位置 + 距我当前位置距离，**以便** 决定是否方便长期合作。
**验收**：
- 仓库卡片含"📍 距您 8.3 km"标注（基于 WA 当前/默认位置）
- 点击进入地图详情页：仓库位置点 + 我位置点 + 路径距离
- 已入驻仓库标"已入驻"标识
- 仅展示 TA 已公开坐标的仓库

### US-OPS-01b · P0 · 平台运维代客创建仓库租户
**作为** 平台运维，**我希望** 直接在后台代客创建仓库租户（无需走 TA 自助注册流程），**以便** 服务地推 BD 拓客、不熟悉自助流程的传统仓库老板、试点仓库快速开通等场景。

**字段与流程**：
- OPS 录入仓库基础信息（名称/地址/营业时间/联系电话）
- OPS 录入租户管理员的手机号（成为初始 TA 账号）
- OPS 上传资质照片（可代收线下扫描件）
- 可选填：是否需要 BD 备注、来源渠道
- 提交后状态直接为「已通过」（OPS 操作天然已审核），跳过 US-OPS-01 排队
- 系统自动向 TA 手机号发送短信：「您的仓库租户已开通，登录链接 / 临时密码 / 首次登录请改密码」
- 自动记录创建人 = 当前 OPS user_id，便于后续溯源

**与自助注册的差异**：
| 维度 | 自助注册 (US-TA-01) | OPS 代建 (本故事) |
|---|---|---|
| 入口 | TA 公开注册页 | OPS 后台「租户管理 → 新建租户」按钮 |
| 审核状态 | 待审核 → 待 OPS 审 | 创建即"已通过" |
| 初次登录 | TA 自设密码 | 短信发临时密码 + 强制首登修改 |
| 资质上传 | TA 上传 | OPS 代上传 |
| 来源标记 | self_register | ops_created (含 ops_user_id) |

**验收**：
- OPS 后台「租户管理」页有「新建租户」按钮（仅 OPS 角色可见）
- 必填字段：仓库名称、地址、联系人手机号
- 选填字段：营业时间、资质照片、备注、来源渠道
- 提交后租户立即可用，TA 手机号收到短信
- TA 首次登录强制走「修改密码」流程
- 操作日志记录：created_by_ops_user_id + created_at + 渠道
- 不影响 US-OPS-01 队列（两条路径并行）

### US-OPS-01c · P0 · 平台运维代客创建批发商账号
**作为** 平台运维，**我希望** 在租户管理员授权或客诉场景下，代客创建批发商（WA）账号入驻某仓库，**以便** 解决批发商不会自助注册或客诉仲裁后需重建账号的场景。

**前置条件**：
- 必须有目标租户的 TA **书面或线上授权**（系统留痕）
- 或基于客诉单 ID 联动（仲裁场景）

**字段与流程**：
- 选择目标租户
- 录入 WA 基础信息（商户名/联系人/手机号/资质）
- 提交后入驻关系直接 = 正常
- 系统短信通知 TA + WA：「OPS 已代建批发商账号」
- 日志记录 ops_created + 授权依据（TA 授权 ID 或客诉单 ID）

**验收**：
- 缺失 TA 授权时按钮置灰
- 创建后 TA 端「入驻批发商」列表新增该 WA，标"OPS 代建"标识
- 系统自动给 TA 推送通知确认

### US-OPS-02 · P0 · SPU 标品库维护
**作为** 平台运维，**我希望** 新增/合并/下架平台 SPU，**以便** 所有仓库的批发商共用统一标品。
**验收**：新增 SPU 1 分钟内对全平台可见；合并 SPU 时，所有引用此 SPU 的 SKU 自动指向新主 SPU 且历史数据保留。

### US-OPS-03 · P0 · 批发商跨租户黑名单
**作为** 平台运维，**我希望** 把违规批发商加入黑名单，**以便** 阻止其在任何仓库重复入驻。
**验收**：黑名单生效后，该手机号/营业执照号不能在任何仓库提交新入驻申请。

### US-OPS-04 · P1 · 平台公告
**作为** 平台运维，**我希望** 发布平台公告，**以便** 通知所有租户/批发商系统升级、政策变更。
**验收**：公告发布后，目标角色登录首页看到弹窗，确认后不再弹。

### US-OPS-05 · P1 · 客诉仲裁
**作为** 平台运维，**我希望** 受理 RT/WA 提交的客诉单，调取相关订单/单据/录音，**以便** 仲裁纠纷。
**验收**：受理后 24h 内出仲裁结论，双方收到结论短信。

## 2. 租户管理员（TA · 仓库老板）

### US-TA-01 · P0 · 注册即开租户
**作为** 仓库老板，**我希望** 用手机号注册并填写仓库资质，**以便** 一步开通租户。
**验收**：注册成功立即创建租户 + 默认店铺；资质审核中状态下可登录但功能受限（仅可填资料）。

### US-TA-02 · P0 · 生成店铺码
**作为** 仓库老板，**我希望** 生成本店的二维码（店铺码），**以便** 终端扫码进店浏览。
**验收**：码可下载/打印；扫码后直接进入本店店铺页。

### US-TA-03 · P0 · 生成员工注册码
**作为** 仓库老板，**我希望** 给库管员/结算员生成专属注册码（可设过期、使用次数），**以便** 防止外人冒充员工注册。
**验收**：注册码扫码后跳转员工注册页，自动绑定本租户；过期/超次数码失效并提示。

### US-TA-04 · P0 · 设置计费规则
**作为** 仓库老板，**我希望** 设置仓储费单价（**件·天 / 托盘·天** 两种维度可任选其一或并存），并配置临期阈值（如保质期剩余 ≤ 30 天预警），**以便** 后续系统自动累计账单。
**验收**：保存后立即对新入库批次生效；历史批次按存入时的规则继续累计。

### US-TA-05 · P0 · 审批批发商入驻
**作为** 仓库老板，**我希望** 查看批发商入驻申请，通过/驳回/设置入驻条件，**以便** 控制店内批发商质量。
**验收**：通过后批发商可在本店上架 SKU；驳回需填理由。

### US-TA-06 · P0 · 内部角色兼任
**作为** 小仓库老板，**我希望** 同一手机号同时担任租户管理员 + 库管员 + 结算员，**以便** 一人即可完成所有操作。
**验收**：兼任后该账号在 3 类角色页面间无缝切换；权限取并集；操作日志按实际操作的角色身份记录。

### US-TA-07 · P0 · 创建自营批发商

### US-TA-12 · P0 · 批次管理开关
**作为** 仓库老板，**我希望** 在店铺设置中维护「批次管理」总开关（默认关闭），**以便** 根据本店运营复杂度决定是否启用批次+保质期跟踪。

**两档含义**：
| 档位 | 含义 | 业务影响 |
|---|---|---|
| 关闭（默认） | 库存仅按 SKU 维度记账，不录批次/保质期 | 入库只填件数+托盘数；无临期预警；FIFO 退化为按入库时间 |
| 启用 | 库存按 SKU+批次维度记账，必录批次号+保质期 | 完整 FIFO + 临期预警 + 批次溯源 |

**切换规则**：
- **关 → 启**：系统为每个有库存的 SKU 自动生成一个「默认批次」占位（批次号 = `DEFAULT-{YYYYMMDD}`，保质期 = 空，可由 WA 后续补录），保留 FIFO 顺序；新入库走正常批次流程
- **启 → 关**：已有批次保留独立库存状态，按老 FIFO 顺序出完为止；新入库不再生成批次，库存合并到 SKU 总数；临期预警停用；展示图"入库实拍"按"最新一次入库"取（无批次概念）

**与其他规则联动**：
- 拍照开关、计费维度、容量公示等不受批次开关影响
- 批次关闭时：盘点单只盘 SKU 总数；不区分批次盘盈/盘亏；US-WK-03 的"批次归属"字段隐藏；临期预警停用
- 批次关闭时 FIFO 退化为"按入库时间先入先出"

**验收**：
- TA 店铺设置页一键切换，关→启需弹窗确认（"将为现有库存生成默认批次占位"）；启→关需弹窗确认（"已有批次将保留出完为止"）
- 切换不影响进行中的入库/出库单（按提交时的策略走完）
- 切换 1 分钟内全店生效；操作日志留痕

### US-WA-10 · P0 · 价格管理（公开价 + 客户专属价）
**作为** 批发商管理员（或被授权员工），**我希望** 有独立的「价格」管理模块，**以便** 集中维护本商户所有 SKU 的公开价与客户专属价，应对老客户复购按议价执行的场景。

**两类价格**：

| 类型 | 适用范围 | 优先级 | 沉淀来源 |
|---|---|---|---|
| 公开价 | 所有 RT 默认看到 | 兜底 | WA/WE 手动维护：单价、起批价、起批量 |
| 客户专属价 | 仅特定 (RT 手机号, SKU) 二元组 | 高于公开价 | 议价确认时沉淀 / WA 手动新增 / 批量调价 |

**价格匹配优先级（RT 浏览/询价时）**：

```
找客户专属价(RT 手机号, SKU, 有效期内 且 未失效)
  ├─ 命中 → 用客户专属价
  └─ 未命中 → 用 SKU 公开价（按起批量判定单价/起批价）
```

**未登录的 RT 仅看公开价**；登录后页面价格自动切到该手机号的专属价（若有）。

**核心能力（P0）**：
- 价格列表页：本商户所有 SKU，每行显示「公开价 + 专属价数量」
- 专属价管理子页：按 SKU 下钻看所有挂在该 SKU 上的客户专属价（手机号 / 单价 / 有效期 / 沉淀来源 / 创建时间）
- 批量调价入口（核心）：
  - 入口 A：整列调价 — 选中 N 个 SKU + 调价方式（涨/降 X% / 改为 ¥X / 加减 ¥X），可单独勾选「同步专属价」
  - 入口 B：按客户调价 — 选某 RT 手机号 → 该手机号下所有专属价批量调
  - 入口 C：手动新增 — 直接为某 (手机号, SKU) 创建专属价（不必走议价）
- 失效操作：单条/批量「立即失效」或「设过期日」
- 调价历史：每次调价留痕（操作人、时间、调前值、调后值、影响条目数）

**议价沉淀逻辑**（与 US-WA-06 联动）：
- 议价回复 / 确认意向单时，价格字段若与公开价不同 → **弹窗询问"是否将本次议价价格沉淀为该客户专属价？"**
- 沉淀选项：本次一次性 / 沉淀为专属价（默认永久）/ 沉淀并设有效期 N 天
- 默认勾选"沉淀为永久"
- 沉淀来源标记为 `from_inquiry`，关联原询价单号便于追溯

**WA 调整公开价的联动**：
- 公开价变更**不影响**已存在的专属价
- 价格管理页对应 SKU 行会显示「公开价变动 X% · 已有 N 条专属价仍生效 [批量复查]」提示
- 点[批量复查]进入"快速调价"页：批量勾选要不要跟随公开价同步调整

**RT 端体验**：
- 商品详情页：登录后看到的价格就是匹配后的最终价；若是专属价，加小标识"专属优惠"
- 我的意向单复购：自动按当前匹配规则取价

**验收**：
- 价格管理菜单：WA 端「价格」一级入口
- 专属价匹配延迟 ≤ 200ms
- 议价确认弹窗 100% 出现沉淀选择
- 批量调价单次最多 500 条；调价历史 1 年内可查
- WA 调整公开价后，1 分钟内提示"已有 N 条专属价仍生效"

### US-WK-03 · P0 · 库存盘点
**作为** 库管员，**我希望** 盘点实物，记录差异并提交"盘盈单"或"盘亏单"，**以便** 系统库存与实物一致，且变动有完整审计链。

**按批次开关分支**：
- 批次启用：按 SKU+批次盘点，盘盈必填批次号+保质期，盘亏必填指定批次
- 批次关闭：按 SKU 总数盘点，盘盈/盘亏不区分批次（盘盈合并到 SKU 总库存）

**差异处理方式**：
- **盘盈/盘亏单 = 独立流水类型**，新增一条流水触发库存数变化，**不修改原入库流水**（保留审计链）
- **全部差异均需 TA 审批**（无自动通过阈值），生效后库存与计费同步联动
- **计费联动**：
  - 盘亏件按盘点日截止计费（次日不再累计）
  - 盘盈件按盘点日起算计费（视同当日入库）
- **不生成财务赔偿单**（遵循"平台不接资金"原则）；盘亏导致的赔偿由 WA 与仓库线下协商

**验收**：
- 提交盘盈/盘亏单 → TA 端"待审批"队列 → 审批通过后库存数变化、流水落库
- 流水类型 = `盘盈` / `盘亏`，与 `入库` / `出库` / `退货` 等流水并列
- 库存数 = 入库流水 - 出库流水 - 退货流水 + 盘盈流水 - 盘亏流水
- 盘点单留痕：操作人、审批人、原值、实物值、差异、理由、时间戳、可附拍照
- 月度账单包含本月盘盈/盘亏的计费影响明细

**验收**：
- 仅"待受理"状态可撤回；"已受理/已入库"不可撤
- 撤回后状态变"已撤回"，不影响库存、不计入计费
- 撤回需填理由（必填）；库管员端实时刷新队列

### R2 · US-WK-R01 · P0 · 入库驳回
**作为** WK，**我希望** 在现场核对发现货物与申请单不符时驳回入库申请，**以便** WA 重新提交。
**验收**：
- 驳回需选驳回原因（数量/质量/批次不符/其他）+ 填备注，可附拍照
- 驳回后单据状态"已驳回"，WA 收到通知，库存不变
- WA 可基于驳回单一键复制内容创建新申请

### R3 · US-WK-R02 · P0 · 入库登记错误纠正
**作为** WK，**我希望** 在入库登记后 24 小时内发现错登（件数/批次/保质期填错）可发起"纠错单"，**以便** 修正库存。
**验收**：
- 超 24 小时禁止纠正，只能走"库存盘点调整"流程
- 纠错单需 TA 审批通过才生效
- 生效后联动调整：库存数 / 批次表 / 已累计仓储费回滚重算
- 整链路留操作日志（原值、新值、操作人、审批人、时间戳）

### R4 · US-WA-R02 · P0 · 出库申请撤回 / 取消
**作为** WA / WE，**我希望** 在出库申请「待受理」或「已打印未实际出库」状态下撤回，**以便** 中止发货。
**验收**：
- "待受理" → 可直接撤回
- "已打印" → 撤回需 WK 二次确认（避免库管已开始拣货）
- "已出库" → 不可撤回，需走 R5 退货流程
- 若该出库单由意向单自动生成（US-WA-06）→ 撤回时同步把原意向单状态回滚到"待确认"，RT 端收到通知

### R5 · US-WA-R03 · P0 · 入库后退货 / 库存回滚
**作为** WA，**我希望** 对已入库的批次发起"退货单"（整批或部分件数退出仓库），**以便** 处理滞销/换仓。
**验收**：
- WA 提交退货申请 → WK 受理 → 现场出库 → 状态"已退货"
- 库存按"件 + 托盘"扣减；已累计的仓储费按"累计到退货日"截止
- 退货数量不超过该批次当前在库数
- 退货单不影响 RT 的意向单流转

### R6 · US-WK-R03 · P0 · 实际出库错误纠正
**作为** WK，**我希望** 在实际出库登记后 24 小时内发现错登（实际拣货数量/批次与登记不符）可发起"纠错单"，**以便** 修正库存与计费。
**验收**：
- 超 24 小时禁止纠正
- 纠错单需 TA 审批；生效后联动调整库存、批次、计费截止时间
- 操作日志留痕

### R7 · US-RT-R01 · P0 · 终端取消意向单
**作为** RT，**我希望** 在意向单「已提交」状态下取消（确认前），**以便** 反悔/改主意。
**验收**：
- 仅"已提交"可取消；"已确认/已拒绝"不可
- 取消后状态"已取消"，WA 收到通知
- 取消理由可选（自由填写或选择"暂不需要"等预设）

### R8 · US-WA-R04 · P0 · 已确认意向单作废（联动撤销已生成出库单）
**作为** WA，**我希望** 在意向单已确认但出库单尚未实际出库时，可作废意向单并联动撤销自动生成的出库单，**以便** 处理客户临时反悔。
**验收**：
- 仅当关联出库单处于"待受理/已打印"状态可作废；"已出库"则需走 R5
- 作废需填理由 + RT 端通知
- 联动：意向单→"已作废"，出库单→"已撤销"
- 日志双向引用可追溯

### R9 · US-WA-R05 · P0 · 询价拒绝（含语音拒绝）
**作为** WA / WE，**我希望** 对不合适的询价单一键拒绝并附理由（缺货/价格谈不拢/其他），**以便** 给 RT 闭环反馈。
**验收**：
- 拒绝后状态"已拒绝"，RT 端通知
- 拒绝不生成出库单
- 拒绝理由可作为后续 WA 复盘数据

### R10 · US-ST-R01 · P0 · 账单调整冲销
**作为** ST，**我希望** 对已生成的账单做"冲销"操作（一笔反向调整抵消原条目），**以便** 处理错算/重复算的情况。
**验收**：
- 冲销不删除原条目，而是新增一条"-原金额"的反向条目并备注关联
- 冲销后总账实时更新
- 冲销记录纳入操作日志

### R11 · US-ST-R02 · P0 · 已下发账单撤回
**作为** ST，**我希望** 在已下发但 WA 未确认/未付款的账单上撤回下发，**以便** 修正错发账单。
**验收**：
- 仅"待回款 + 0 回款记录"状态可撤回
- 已部分回款的账单不可撤回，只能调整
- 撤回后状态回"待核对"，WA 端原账单标"已撤回"

### R12 · US-ST-R03 · P0 · 已收款登记冲销
**作为** ST，**我希望** 在误登记或退款时冲销已收款记录，**以便** 修正回款状态。
**验收**：
- 冲销需填理由（如"误登""客户退款"），可附凭证
- 冲销后账单回到"待回款"或"部分回款"状态
- WA 端收到通知

### R13 · US-WA-R06 · P0 · 批发商主动退驻
**作为** WA，**我希望** 在结清所有账单 + 清空库存后申请退出仓库，**以便** 终止入驻关系。
**验收**：
- 前置条件：库存 = 0，账单已结清，无未结意向单/出库单
- 提交后 TA 审批；通过则账号转"已退驻"，SKU 全部下架、店铺页隐藏本商户
- 60 天内可恢复（保留历史数据），逾期归档

### R14 · US-TA-R01 · P0 · 批发商被动下架 / 封禁
**作为** TA，**我希望** 对违规批发商单方面下架（含未结清账单的强制下架），**以便** 处理纠纷。
**验收**：
- 下架立即生效：店铺页隐藏、SKU 不可购买、新询价/出库被拒
- 已生成的出库单 / 已确认意向单允许继续完成（保护客户权益），但不再接新业务
- 未结账单标"争议中"，由 OPS 客诉仲裁

### R15 · US-OPS-R01 · P0 · 平台冻结 / 解冻租户
**作为** OPS，**我希望** 在租户严重违规时冻结整个租户（含 TA / WK / ST / 入驻 WA），**以便** 全平台风控。
**验收**：
- 冻结：禁止登录、店铺码失效、所有单据流转暂停
- 解冻：恢复所有权限，数据不丢
- 操作需双人复核（OPS 主管二次确认）

### R16 · US-RT-R02 · P0 · 终端账号注销
**作为** RT，**我希望** 注销账号删除我的手机号/历史询价，**以便** 保护隐私。
**验收**：
- 注销前展示"7 天冷静期"，期间可撤销注销
- 7 天后手机号脱敏（不可登录），询价单保留但隐去 RT 身份（保留 WA 业务记录完整性）
- 符合《个人信息保护法》注销权要求

### R17 · US-TA-R02 · P0 · 员工离职 / 权限回收
**作为** TA / WA，**我希望** 一键禁用某员工账号（库管/结算/批发商员工），**以便** 离职即时回收权限。
**验收**：
- 禁用后该员工立即无法登录，正在编辑的草稿单据自动作废
- 已提交的单据仍归属原员工，操作日志不变
- 禁用可撤销（30 天内）；30 天后彻底删除账号

### R18 · US-WA-R07 · P0 · 语音下单录音作废
**作为** WA / WE / RT，**我希望** 在语音识别预填后、提交前主动放弃整段录音并清空表单，**以便** 重新录入。
**验收**：
- "放弃"按钮位于语音预填表单顶部
- 放弃后清空所有预填字段（包括备注）
- 录音文件按 01 §6.5 留 30 天再删除（不因放弃即时删，避免取证缺失）

### R19 · US-WK-R04 · P0 · 临期商品强制处理
**作为** WK + TA，**我希望** 对保质期归零的批次发起"强制清库"，**以便** 避免过期货物长期占库。
**验收**：
- 系统每日扫描，到期批次自动标"待清理"
- WK 发起清库单 + 拍照凭证 → TA 审批 → 库存清零、计费截止
- WA 端站内信通知（含清理凭证照片）

### R20 · US-ST-R04 · P0 · 计费规则变更回溯保护
**作为** ST / TA，**我希望** TA 修改计费规则时不影响已发生的历史账单，**以便** 保持账单严肃性。
**验收**：
- 规则变更只对未来日期生效（含本月剩余天数）
- 已生成的历史账单不重算；当月在算账单按"分段计费"（变更日前用旧规则，变更日后用新规则）
- 变更需 TA 二次确认 + 给所有入驻 WA 推送通知

## 8. 关键场景串联

### S1 · 完整入库链路（WA 提交 → WK 入库 → ST 计费起算）
```
WA: 提交入库申请（含批次/保质期/占用托盘数）
 → WK: 现场核对 → [可选] 打印入库单供签收
 → WK: 拍照存证（≥1 张）
 → WK: 登记实际入库
 → 库存 + 批次表更新
 → 照片入"入库相册"按批次分组
 → SKU 自动转"上架"（除非之前手动下架）
 → 入库单可补打留底（盖章存档）
 → 系统按"件·天 + 托盘·天"开始累计仓储费
 → 月底 ST 收账单
（若 WA 已选"展示图=入库实拍"→ 终端店铺页商品图自动跟随最新批次）
```

### S2 · 询价 → 自动出库链路（RT 询价 → WA 确认 → WK 出库）
```
RT: 浏览店铺 → 选 SKU → 语音/手填提交询价
 → WA: 询价队列 → 议价 → 确认
 → 系统：自动生成出库申请单（带原意向单 ID）
 → WK: 出库队列 → FIFO 拣货 → 打印 → 实际出库
 → 库存扣减 → RT 收到状态变更通知
```

### S3 · 月度账单链路（ST 生成 → WA 接收 → 线下回款 → ST 登记）
```
1 日 0:00: 系统自动生成上月账单
 → ST: 核对 → 调整（如有） → 下发
 → WA: 收到通知 → 查看明细 → 申诉/接受
 → 线下转账（平台不经手资金）
 → ST: 手动登记"已收款"
 → 账单状态变"已结清"
```

### S4 · 一人多岗操作链路（小仓库 TA 兼库管+结算）
```
TA 上午: 入库受理身份 → 登记入库
TA 中午: 询价处理（如自营批发商）
TA 下午: 结算员身份 → 调整账单 → 下发
TA 晚上: 老板身份 → 看店铺日报
全程: 单账号、权限并集、操作日志区分实际角色
```

### S5 · 语音下单兜底链路（识别失败 → 手填）
```
WE 长按麦克风 → 录音 ≤ 60 秒
 → 实时转写 → NLU 抽取字段
 → SKU 模糊匹配失败 → 标红
 → 用户手动选择 SKU + 改件数
 → 确认 → 提交
（连续 3 次失败 → 系统提示切表单录入）
```

### S6 · 意向单 → 出库链路的撤销联动（R8）
```
RT 提交意向单 → WA 确认 → 系统自动生成出库单
 → WK "已打印"（未实际出库）
 → WA 发现客户反悔，发起意向单作废（R8）
 → 系统校验：出库单非"已出库" → 允许联动
 → 意向单 → "已作废"
 → 出库单 → "已撤销"
 → RT + WK 端收到通知；操作日志双向引用
```

### S7 · 入库后退货链路（R5）
```
WA 发起退货申请（指定批次 + 件数）
 → WK 受理 → 现场出货 → 实际登记
 → 库存按件 + 托盘扣减
 → 计费截止到退货日
 → ST 当月账单按"实际占用天数"出账
（若涉及当月新入库的批次 → 计费按"半天/全天"按 04 流程图规则）
```

### S8 · 账单异常处理链路（R10 + R11 + R12）
```
ST 发现错算
 → 已下发？
   是 → 已回款？
     否 → R11 撤回下发 → 改正 → 重新下发
     部分回款 → R10 冲销错项 → 调整 → 重新下发余款部分
   否 → 直接 R10 冲销 + 调整 → 下发
 → 若发现已收款误登
   → R12 冲销已收款 → 账单回到"待回款"
```

### S9 · 批发商退驻 / 被封禁的链路差异（R13 vs R14）
```
R13 主动退驻：
 WA 自查（库存=0 + 账单结清 + 无未结单）
  → 提交退驻 → TA 审批 → 转"已退驻"
  → 60 天内可恢复；逾期归档

R14 被动下架：
 TA 发起强制下架（不等结清）
  → 立即：店铺页隐藏 / 新业务封禁
  → 老业务：已确认意向单 + 已生成出库单允许完成
  → 账单未结 → 标"争议中" → OPS 仲裁
```

### S10 · 库存盘点差异处理链路（US-WK-03）
```
WK 现场盘点 → 录入实物数 vs 系统数
 → 系统计算差异：
   - 实物 > 系统 → 生成"盘盈单"（必填 批次号+保质期）
   - 实物 < 系统 → 生成"盘亏单"（必填 指定扣减的批次号）
 → 提交 → TA 审批队列
 → TA 通过 → 流水落库：类型=盘盈/盘亏
 → 库存数 = 入库 - 出库 - 退货 + 盘盈 - 盘亏
 → 计费联动：
   盘亏件 → 盘点日为截止日（次日不再累计）
   盘盈件 → 盘点日为起算日（视同当日入库）
 → 月度账单包含本月盘盈/盘亏的计费影响行
 → 不生成赔偿单（线下协商）
```
