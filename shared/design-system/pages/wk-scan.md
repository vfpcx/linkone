## 全屏摄像头模式

### 布局

```
┌─────────────────────┐
│ ← 扫码入库      🔦 ⌄│   ← 顶部黑色 50% 透明遮罩
│                     │
│                     │
│   ┌───────────┐     │   ← 扫码框：380×380px，居中
│   │           │     │     白色 2px 描边 + 四角加粗
│   │  扫描区   │     │
│   │           │     │
│   └───────────┘     │
│                     │
│   将条码对准扫描框    │   ← 引导文字，14px 白色
│   自动识别后弹出确认  │
│                     │
│   [手动输入]         │   ← 兜底按钮：白边白字 透明底
│                     │
│ ━━━━━━━━━━━━━━━━━━━│   ← 底部 30% 透明黑遮罩
│ 已扫: 3 件           │
│ 待扫: 17 件          │
└─────────────────────┘
```

### Token 偏离

| Token | MASTER | 此页 |
|---|---|---|
| 背景 | `--color-bg-2` (白) | **#000000**（摄像头底层）|
| 文字 | `--color-fg-1` | **#FFFFFF** + 50% 半透 |
| 顶部栏 | 藏青 | **半透黑** `rgba(0,0,0,0.5)` |
| 扫码框描边 | — | **#FFFFFF 2px** + 四角橙色加粗 `--color-rt-warm` |
| 引导文字 | 14px | **14px 白色 + drop-shadow** |

### 关键交互

1. **进入页 → 立即请求相机权限**，拒绝时显示"无法扫码，请允许相机权限"+ 手动输入
2. **扫码框内激光线动画**：500ms 上下移动，纯 CSS transform，不阻塞 input
3. **扫描成功**：
   - 振动反馈（uni.vibrateShort）
   - 扫描框由白变绿（`--color-success`）持续 300ms
   - 弹出底部弹窗显示扫到的 SKU 信息 + [确认] [重扫] [手动改]
4. **闪光灯按钮**：右上角 🔦，环境光暗时常亮，否则灰显
5. **手动输入按钮**：永远在底部 80px 高位置，单手可达
6. **批量扫码模式**：连续扫码不弹窗，仅 toast "已扫 X 件"

### 弹窗（扫码后）

```
┌─────────────────────┐
│ ✓ 扫到了             │   ← 顶部绿色横条
│                     │
│ SKU: 三文鱼整条 5kg  │
│ 批次号: BATCH-A1     │
│ 保质期: 2027-05-15   │
│                     │
│ 件数 <  20  >        │   ← 默认 1 件，可手动改
│                     │
│ [重扫]  [手动改] [确认入库]│
└─────────────────────┘
```

- 底部弹窗 from bottom，`--shadow-lg`
- "确认入库"按钮主蓝、48px 高、占 50% 宽
- 顶部成功横条 3 秒后自动收起

### 故障与兜底

- ✅ 相机权限拒绝 → 大图标 + 文字 + "去设置" + "手动输入" 双按钮
- ✅ 扫到非本店 SKU → 红色 Toast "该 SKU 不属于本店任何批发商"，3 秒
- ✅ 扫到已有相同批次 → 询问"合并到现有批次 N 件 还是 新建批次"
- ✅ 连续 3 次扫不到 → 提示"切换到手动输入"

## 不允许的偏离

- ❌ 不要装饰性扫码动画（如全息、星光、3D）
- ❌ 不要打开扫码立即扫到就跳出页，必须给确认机会
- ❌ 不要把"手动输入"藏到二级菜单 —— 必须一级可达
