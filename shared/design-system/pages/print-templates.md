# 单据打印模板 · 设计覆写

> 适用：入库单、出库单、退货单、临期清库单 的打印模板
> 覆写自：`../MASTER.md`

## 双规格支持

| 规格 | 适用 |
|---|---|
| **A4 纸（210×297 mm）** | 入库单、出库单（正式存档） |
| **80mm 热敏（80mm 宽，长度自适应）** | 现场出库小票 |

## 通用规则

- 全黑白 / 灰度，不允许彩色
- 字号：A4 用 10-12pt；80mm 用 9-11pt
- 字体：思源宋体 / 等线 / Times New Roman（衬线，老式打印机识别更佳）
- 公司名 + Logo 顶部 + 单号 + 二维码定位
- 所有金额、数字 tabular-nums（即使打印也要对齐）
- 表格用细线 0.5pt 边框
- 不使用阴影、圆角、渐变（打印机解析不稳）

### 必要元素
- 二维码：编码单号，扫描后可在系统查询完整明细

- 合计行加粗 + 上下边框加粗

```
-----------------------------
合计               30   2
=============================
[二维码]

库管员：____  收货人：____

=============================
```

### 字号约定
- 标题：12pt + 加粗
- 表头：10pt + 加粗
- 内容：10pt
- 合计行：11pt + 加粗

## 实现建议

- 用 `window.print()` + `@media print` CSS
- PDF 导出用 jsPDF + html2canvas（前端方案）或后端 wkhtmltopdf
- 热敏小票用 ESC/POS 协议库 `escpos-printer-driver`，或直接生成图片让 thermal printer 渲染

## 不允许

- ❌ 打印模板使用 web 字体（用户机器没有 → fallback 乱）
- ❌ 打印模板里嵌入彩图（除 Logo 和二维码黑白）
- ❌ 打印模板里有交互按钮、链接、动画

## 大额校验 Modal（业务关键）

代建出库（US-WK-02b）件数 > 该 SKU 在库 50% 时强制弹出：

```
┌─────────────────────────────────────┐
│ 🔴  大额出库提醒                      │
├─────────────────────────────────────┤
│ 本次出库 30 件 > 在库 56 件的 50%    │
│ 请再次确认数量是否正确。             │
│                                      │
│ 二次输入件数 <        30        >    │
│ 操作说明 *                           │
│  <____________________________>      │
│                                      │
│      [取消]    [我已核对，确认]      │
└─────────────────────────────────────┘
```

### 偏离 MASTER

- 标题颜色用 `--color-danger` 红
- 顶部红色横条 4px
- 二次输入框 placeholder 显示"请再输入一次件数"
- "我已核对，确认"按钮红底白字，需要前两个字段都填齐才激活
- 大额阈值由计算："本次件数 / 当前在库数 ≥ 0.5"

### 二次输入校验

```js
// 必须严格相等
if (twiceInput !== originalInput) {
  toast.error('两次件数不一致，请重新输入');
  return;
}
if (reason.trim().length < 4) {
  toast.error('请填写操作说明（至少 4 字）');
  return;
}
```

### 日志留痕

提交后操作日志必含：原始件数、二次输入件数、说明、操作时间、IP、设备 UA、当时该 SKU 在库件数（防"事后改库存"绕过校验）。
