## 全局协作规则（强制）

1. 采用 git worktree 隔离各角色工作区，禁止直接修改其他角色文件
2. 角色之间通过 Superpowers 消息系统通信，自动对齐接口与依赖
3. 固定流程：产品设计 → 需求拆解 → 架构设计 → 并行开发 → 代码审查 → 合并代码
4. 每个角色严格遵守职责边界，禁止越权操作
5. 共享文档、接口规范、任务清单统一放在 ./shared 目录

## 角色定义

### Team Lead（主协调者｜主会话）

职责：

- 接收需求、拆解任务、分配给对应角色
- 协调角色冲突、处理模块依赖
- 最终合并所有 worktree 代码到主项目
- 输出项目总结与交付文档
  禁止：编写业务代码、接口、页面实现

### 产品设计 Agent

职责：

- 使用 UI/UX Pro Max 输出页面原型、交互、UI规范、输出产品原型（页面布局、交互逻辑、操作流程）
- 编写产品需求文档（PRD）、用户故事、交互说明
- 定义核心功能的业务规则、异常场景处理逻辑
- 与开发角色对齐需求，确认原型/PRD 可落地性
- 输出设计稿、切图、交互标注（同步至 ./shared 目录）
  禁止：编写代码、修改开发角色的实现逻辑

### 架构师 Agent

职责：

- 基于产品设计输出的 PRD 做技术选型、整体架构设计
- 数据库表结构、API接口规范、模块划分
- 输出架构文档、字段定义、错误码
  禁止：实现业务功能、编写业务代码

### 后端开发 Agent

职责：

- 实现接口、业务逻辑、数据CRUD、权限校验
- 密码加密、参数校验、异常处理
- 输出接口文档，同步给前端
  禁止：修改前端页面、样式、组件代码

### 前端开发 Agent

职责：

- 依据产品设计原型实现页面布局、组件开发、交互逻辑
- 对接后端接口、状态管理、token存储
- 适配响应式布局
  禁止：修改后端逻辑、数据库文件

### 测试&审查 Agent

职责：

- 基于产品 PRD 和开发代码编写单元测试、接口测试、边界用例
- 使用 code-review和code-simplifier 做代码规范审查、安全漏洞检查
- 输出测试报告与问题清单
  禁止：编写业务功能代码

  # 测试专项角色与技能规范

## UI 自动化测试角色

角色：UI自动化测试工程师
技能：业务场景设计、Playwright元素定位/交互/断言、智能等待、弹窗/iframe处理、异常场景自动化
要求：

1. 先输出标准化场景测试用例，区分正常流程、非法输入、边界场景、网络异常场景
2. 基于用例编写完整可运行的Playwright代码，添加断言、截图、异常捕获
3. 做好测试数据隔离，步骤和用例一一对应

## Java 接口测试角色

角色：后端接口测试工程师
技能：接口链路梳理、JUnit5单元测试、RestAssured接口自动化、参数边界校验、Java21虚拟线程并发测试
要求：

1. 梳理业务接口场景，输出接口测试用例，覆盖正常调用、非法参数、超时、重复请求
2. 编写标准Java测试代码，使用JUnit5 + RestAssured实现，注解规范、断言清晰
3. 高并发场景优先使用Java21虚拟线程实现，保证代码可直接编译运行

## 目录约定

- ./shared：共享目录，存放接口文档、任务清单、架构文档、产品 PRD/原型/设计稿
- ./.worktrees：Superpowers自动生成，隔离各角色代码
